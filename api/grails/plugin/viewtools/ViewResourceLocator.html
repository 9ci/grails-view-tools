Package: grails.plugin.viewtools

[Groovy] Class ViewResourceLocator

    • grails.plugin.viewtools.ViewResourceLocator
    • All Implemented Interfaces and Traits:
      ResourceLoader, ResourceLoaderAware, ApplicationContextAware, PluginManagerAware

      @SuppressWarnings(['PublicInstanceField', 'NonFinalPublicField', 'UnnecessarySubstring'])
@Slf4j
@CompileStatic
class ViewResourceLocator
extends Object

      Extended because the handy methods we want in DefaultGroovyPageLocator are protected. Used to locate View resources whether in development or WAR deployed mode from static resources, custom resource loaders and binary plugins. Loads from a local grails-app folder for dev and from WEB-INF in development mode. GroovyPageResourceLoader bean exists in dev/test mode and deals with plugin paths, inplace etc... will not be here in deplyed war

      See Also:
      GroovyPagesGrailsPlugin is where the original beans are setup, take a look at the source to see how they are setup
      heavily as a starting point
